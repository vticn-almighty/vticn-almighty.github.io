# 问题出现

当我们进行网页开发时可能不需要使用样式库中的样式，例如disabled的样式(在按钮按下激活的程序失效的时候使用按钮禁用避免用户入坑)，这时候我们载入bootstrap的样式文件同时就载入了我们不需要的代码，我在此只举了bottom这一个例子，要知道页面中还有很多元素，bootstrap也为此定制了很多样式，再加上媒体查询，我们实际用到的只是其中的一小部分，大部分的冗余代码将影响加载网页的速度

# 解决方案

我们可以采取以下应对措施以去除这些多余的样式定义:

  • 去网站在线处理
  • 使用webpack插件处理
  • 使用grunt-uncss处理

第一个网站提供uncss的解决方案，需要将css和html代码复制粘贴到对应输入框中，然后又需要复制生成的css代码，它可以完美解决冗余css去除的难题，甚至可以解决媒体查询中的冗余，但是处理速度慢，自动化程度低。

第二个方案需要配置一定的webpack配置文件，但是不能很好的解决样式库中媒体查询的冗余问题，去冗余处理后依然存在大量代码冗余，是亟待完善的方案，由于webpack是主流包处理工具，所以这个方案完善后能更好的配合webpack强大的生态，自动化程度将大幅提升

第三个方案弥补了css媒体查询和方案一需要手动复制粘贴的问题，处理速度快，依赖命令行进行，但是本方案本身依赖grunt，可能无法跟现在某些主流模块很好的配合

# 测试最终方案

Gruntfile.js是grunt的配置文件，内容如下:

'use strict';

module.exports = function (grunt) {
    require('load-grunt-tasks')(grunt, { scope: 'devDependencies' });
    require('time-grunt')(grunt);

    // Project configuration.
    grunt.initConfig({
        // Before generating any new files, remove any previously-created files.
        clean: {
            tests: ['tmp', 'dist', 'tests/output.css']
        },

        uncss: {
            dist: {
                src: ['tests/app/about.html', 'tests/app/contact.html', 'tests/app/index.html'],
                dest: 'dist/css/tidy.css'
            },
            test: {
                files: {
                    'main.css': 'tests/index3.html'
                },
                options: {
                    report: 'gzip'
                }
            },
            testMany: {
                files: {
                    'tests/output.css': 'tests/index.html',
                    'tests/output2.css': 'tests/index2.html'
                },
                options: {
                    report: 'gzip'
                }
            },
            testUncssrc: {
                files: {
                    'tests/output.css': 'tests/index.html'
                },
                options: {
                    uncssrc: 'tests/.uncssrc'
                }
            },
            testUrl: {
                files: [{
                    nonull: true,
                    src: ['https://getbootstrap.com/docs/3.4/examples/jumbotron/'],
                    dest: 'tests/outputUrl.css'
                }]
            }
        },

        processhtml: {
            dist: {
                files: {
                    'dist/about.html': 'tests/app/about.html',
                    'dist/contact.html': 'tests/app/contact.html',
                    'dist/index.html': 'tests/app/index.html'
                }
            }
        },

        cssmin: {
            dist: {
                options: {
                    compatibility: 'ie8',
                    keepSpecialComments: 0
                },
                files: {
                    '<%= uncss.dist.dest %>': '<%= uncss.dist.dest %>'
                }
            }
        },

        copy: {
            dist: {
                files: [{
                    expand: true,
                    cwd: 'tests/app/',
                    src: ['img/**', 'js/**', '*.png', '*.xml', '*.txt', '*.ico', '!*.html'],
                    dest: 'dist/'
                }]
            }
        },

        // Unit tests.
        simplemocha: {
            test: {
                src: 'tests/selectors.js'
            }
        },

        connect: {
            options: {
                hostname: 'localhost',
                livereload: 35729,
                port: 3000
            },
            livereload: {
                options: {
                    base: 'dist/',
                    open: true // Automatically open the webpage in the default browser
                }
            }
        },

        watch: {
            options: {
                livereload: '<%= connect.options.livereload %>'
            },
            files: ['Gruntfile.js', 'tasks/**/*.js', 'tests/**/*.*'],
            tasks: ['test']
        }

    });

    // Actually load this plugin's task(s).
    grunt.loadTasks('tasks');

    grunt.registerTask('test', [
        'uncss:test',
        // 'uncss:testMany',
        // 'uncss:testUncssrc',
        // 'uncss:testUrl',
        // 'simplemocha',
        // 'default'
    ]);

    grunt.registerTask('dev', [
        'test',
        'connect',
        'watch'
    ]);

    // By default, lint and run all tests.
    grunt.registerTask('default', [
        'clean',
        'copy',
        'uncss:dist',
        'cssmin',
        'processhtml'
    ]);
};

其中'tests/output.css'指明我们去冗余处理后的css文件将在tests目录下以output.css的文件名输出，'tests/index.html'指明去冗余的依据以及css文件依赖，grunt-uncss需要根据html文件中依赖的css文件，以及html代码本身去冗余

我们可以看到，index.html载入了多条css文件，grunt-uncss会从index.html文件依赖的这些css文件中读取内容，然后根据body标签中的元素id，class或者标签名以及元素关系过滤掉css文件中存在但未在index.html的主体(body)中使用的部分

让我们回归Gruntfile.js，其中定义了report: 'gzip'这个选项(option)，它在grunt中存在两种配置，分别为'gzip'和'min'，这将告诉grunt如何汇报文件大小的处理结果

我们对比一下两种配置的显示结果:

min gzip
File tests/output.css created: 1.6 kB → 1.19 kB
File tests/output.css created: 1.6 kB → 1.19 kB → 404 B (gzip)

可见gzip配置会显示css去冗余后在服务器的gzip压缩后的文件大小，由于是需要在服务器端才能这么做，所以我们在这个测试中实际上看不到压缩为gzip后的结果的(服务器端优化为output.css.gz这样的文件)

# 总结

grunt-uncss很好的根据html-css的依赖关系快速的消除冗余css代码，但是该方案基于grunt，对于webpack爱好者来说无法使用。webpack提供的模块中又不容易解决css媒体查询中的冗余问题。

grunt-uncss提供的测试任务中还有一部分内容未在本文介绍，但是原理都是基于html-css的关系进行的，有兴趣的同学也可以自行测试。本作者在文章中提到的部分完全可以完成去冗余任务需要。
